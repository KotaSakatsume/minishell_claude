## PR Body

### Summary
入出力リダイレクトを実装し、flat な `argv` から **型付きトークン列 + パーサ + `t_cmd { argv; redirs }`** へ移行する(Issue #10)。**`< > >> <<`(heredoc)が外部コマンドにもビルトインにも効く。**

- **レキサ:** 型付きトークン(WORD / IN / OUT / APPEND / HEREDOC)を生成。`< > >> <<` は**クォート外でのみ**演算子化(`echo ">"` はリテラル)。`>file`(空白なし)も語境界
- **パーサ:** トークン列 → `t_cmd`(出現順 redir リスト)。`>` の後に語が無ければ syntax error(status 2)
- **適用:** 外部=子で `dup2`、ビルトイン=親で `dup` 退避 → 適用 → 復元。フラグ O_TRUNC/O_APPEND、open 失敗の errno→bash 風メッセージ(ENOENT/EACCES/EISDIR)+ status 1 非実行
- **heredoc:** pipe 方式で delimiter まで読み読み端を stdin に。**展開しない / history に積まない**
- **redir のみ**(`> f`)はファイル作成/truncate、status 0

### Verification(すべて実測 / Intel Mac・Apple clang 14・GNU Make 3.81)
- ✅ `echo hi > f; cat < f` / `>>` append / `pwd > f; pwd`(builtin 退避復元で端末復帰)
- ✅ `cat << EOF` heredoc(history 非汚染)/ `cat << A << B`(最後が stdin)
- ✅ `echo ">"`→リテラル、`echo x>y`→分離
- ✅ open 失敗 ENOENT/EACCES/EISDIR → メッセージ + status 1 非実行
- ✅ `ls >` / 単独演算子 → syntax error status 2(クラッシュなし)
- ✅ 複数 redir 最後が有効 / `> f` 空作成 status 0
- ✅ 回帰: `ls`/`pwd`/`cd`/`export`/`unset`/クォート/`$VAR`/`$?` 不変
- ✅ `make re` 警告ゼロ・二度目再リンクなし
- ✅ メモリリーク **0**(非フォーク経路: parser/redir-only/builtin-redir/構文・parse エラー)
- ✅ **fd リークなし**(`/dev/fd` が bash と同一)
- ✅ norminette: 42 ヘッダー以外の違反ゼロ

### Known / Deferred
- **42 ヘッダー未付与**(ログイン待ち)。
- **`echo hi > $NOPE`**(target が空展開)→ "syntax error" status 2(bash は "ambiguous redirect" status 1)。#8 の空展開除去と地続き。エラーで非実行・非ゼロ status になる点は妥当。
- **heredoc:** 変数展開なし(最小固定)/ 大容量(pipe バッファ ~64KB 超)はスコープ外 / Ctrl-C 中断はシグナルIssue。
