## PR Body

### Summary
空白 split トークナイザを **1パスの quote-aware レキサ + インライン変数展開**に置き換える(Issue #8)。**`'...'` / `"..."` のクォート処理と `$VAR` / `$?` の展開が動く。**

- **シングルクォート** = 中をリテラル(展開なし)、**ダブルクォート** = `$` 以外リテラル($展開あり)、クォートは除去、隣接断片は連結(`a"b c"d` → `ab cd`)
- **`$VAR`**(`env_get`、未定義は空)、**`$?`**(`last_status`)。`$` の後が名前でなければリテラル `$`。`$1`(数字始まり)はリテラル(位置パラメータ非対応)
- **unquoted の空展開語は除去**(`$NOPE` → no-op)、**quoted 空語は保持**(`""` / `"$NOPE"` → 空引数)= bash の引数個数に一致
- **可変長バッファ**は倍化 malloc + コピー + free の自前 grow(**realloc は許可関数外**)
- **未閉じクォート** = エラー表示 + 行破棄 + `last_status=2`
- `tokenize(shell, line)` に変更(env/`$?` 参照のため)

### Verification(すべて実測 / Intel Mac・Apple clang 14・GNU Make 3.81)
- ✅ `echo '$HOME'`→リテラル、`echo "$HOME"`/`$HOME`→値、`echo $?`→ステータス、`echo a"b c"d`→`ab cd`、`echo a""b`→`ab`、`export X=hi; echo $X`→`hi`、`echo "hello"`→`hello`
- ✅ `$NOPE`→no-op、`echo "$NOPE"`→空引数保持(bash 引数個数一致)
- ✅ 未閉じクォート→エラー + `$?`=2 + 行破棄
- ✅ 回帰: `ls`/`pwd`/`cd`/`export`/`unset`/`env`/`exit` 従来どおり
- ✅ `make re` 警告ゼロ・二度目再リンクなし
- ✅ `leaks` 自作リーク **0**(grow 複数回・展開・空語除去・未閉じエラー経路)
- ✅ norminette: 42 ヘッダー以外の違反ゼロ(関数25行 / 5関数 / while のみ / グローバル0個 / 80列)

### Known / Deferred
- **42 ヘッダー未付与**(ログイン待ち)。
- **`$1`(数字始まり)= リテラル**(位置パラメータ非対応の必然。bash は空)。
- **タブがクォート内で保持されない** — readline が非対話パイプ入力で TAB を消費(レキサに届く前)。レキサ自体は空白を保持する(確認済み)。
- **フィールド分割なし**(`X="a b"; echo $X` は1引数)。

### Scope(やらないこと)
