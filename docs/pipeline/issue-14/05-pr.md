## PR Body

### Summary
シグナル処理を実装し、唯一許可されるグローバル変数 `sig_atomic_t g_signal`(受信番号のみ)を導入する(Issue #14)。**これで 42 Minishell マンダトリーの主要機能が一通り揃う。**

- **新規 `signals.c`:** 対話モード(SIGINT→新プロンプト `rl_on_new_line`/`rl_replace_line`/`rl_redisplay`、SIGQUIT 無視)/ 実行親(SIGINT/SIGQUIT を SIG_IGN=シェル生存)/ 子(SIG_DFL=実行コマンドが死ぬ)
- **`repl.c`:** 反復先頭で対話シグナル設定、`g_signal==SIGINT` を `last_status=130` に変換(ハンドラは t_shell に触れず番号のみ=42 規約)
- **`process_line.c`:** exec 前に親を SIG_IGN(単一 + パイプ)。`child_run_external`/`pipe_child` が子を SIG_DFL に
- **`wait_status`** は既存で 128+sig → 130(INT)/ 131(QUIT)

### Verification(実測 / Intel Mac・Apple clang 14。対話は pty = expect + python)
- ✅ 空プロンプト ctrl-C → 新プロンプト + `$?`=**130**、入力途中 ctrl-C → 行破棄
- ✅ `sleep` 実行中 ctrl-C → 中断・`$?`=**130**、ctrl-\ → Quit・`$?`=**131**、シェル生存
- ✅ 対話プロンプト ctrl-\ → 無視・生存、ctrl-D → 終了
- ✅ 回帰: パイプ / リダイレクト / builtin / クォート / `$`展開 不変
- ✅ **ファイルスコープのグローバルは `g_signal` ただ1個**(番号のみ、t_shell 非アクセス)
- ✅ ハンドラ async-signal-safe(printf/malloc/free 不使用)
- ✅ `make re` 警告ゼロ・二度目再リンクなし / メモリリーク 0(非フォーク)/ norminette は 42 ヘッダー以外クリーン

### Known / Deferred
- **42 ヘッダー未付与**(ログイン待ち)。
- **heredoc 中 ctrl-C はスコープ外** — `run_heredocs` が親 SIG_IGN 後に走るため heredoc 入力中の ctrl-C は無視(bash は中断)。SA_RESTART 一時解除/別ハンドラが要り複雑 → 後続Issue。コマンドプロンプト + 実行中断は確実に満たす。
- **親 builtin 実行中の ctrl-C は無視**(実害小)/ 中断時の追加改行なし(端末依存)。

### Scope(やらないこと)
ジョブ制御 / ctrl-Z(SIGTSTP)/ `setpgid` / `^C` エコー抑制(termios)/ `&&`/`||` は本PR対象外(別Issue)。
