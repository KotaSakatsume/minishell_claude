## PR Body

### Summary
`|` で複数コマンドを接続するパイプラインを実装する(Issue #12)。`t_cmd` を `next` 連結リスト化し、各段を fork して pipe で接続する。**マンダトリーで残るはシグナルのみ。**

- **レキサ:** `|` をクォート外で `TOK_PIPE` 化(`||` は1文字ずつ、`echo "|"` はリテラル)
- **パーサ:** トークンを `|` で分割し `t_cmd.next` リスト構築。空の段(argv も redirs も無い)/ 末尾 `|` は syntax error(status 2)
- **実行:** 単一コマンドは**既存の親実行経路を温存**(cd/export/exit がシェルに効く)。パイプラインは各段を fork、N-1 本 pipe で stdin/stdout 接続(親は書端を fork 直後 close、前段読端を持ち回り)、**全子 waitpid(ゾンビ無し)で最終段の status** を採用
- **パイプ内 builtin は子で実行 → exit**(cd/export がシェルに効かない=bash 同等)。redir/heredoc は pipe dup2 後に適用(優先)
- `execute.c`: `child_exec` を `exec_external`(execve のみ)に分割、redir は呼び出し側で適用、単一/パイプ子で共有

### Verification(すべて実測 / Intel Mac・Apple clang 14・GNU Make 3.81)
- ✅ `echo hello | cat` / `ls | wc -l` / 3〜5段チェーン / grep チェーン / `seq 1 1000 | wc -l`(throughput)
- ✅ redir 優先 `echo hi | cat > out` / 先頭段入力 redir `cat < f | wc -l` / redir-only 段 `> a | echo hi`
- ✅ 最終段 status(`false|true`=0, `true|false`=1)/ `exit | cat` でシェル生存 / `nosuchcmd | echo` で last=0
- ✅ `echo "|"` リテラル / syntax error(`|cat`/`echo|`/`a||b`)=status 2
- ✅ 回帰: 単一 builtin の親実行(cd/export/exit がシェルに反映)/ redirect/quote/`$`展開 不変
- ✅ `make re` 警告ゼロ・二度目再リンクなし
- ✅ メモリリーク **0**(parse 経路)/ fd リークなし(`/dev/fd` が bash 同一)/ **ゾンビ 0**(稼働中 minishell の defunct 子 0)/ デッドロックなし
- ✅ norminette: 42 ヘッダー以外の違反ゼロ
