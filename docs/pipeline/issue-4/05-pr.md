## PR Body

### Summary
`process_line()` のエコー実装を、実際にコマンドを実行する単一コマンド実行パイプに置き換える(Issue #4)。**`ls`(外部)と `pwd`(ビルトイン)が動く。**

- **トークン化**: 入力行を空白区切りで `argv` に分割(クォート/展開/リダイレクト/パイプは別Issue)
- **外部コマンド**: `shell->envp` から PATH を探索、`fork`/`execve`/`waitpid`。終了コードを `last_status` に保存(`ENOENT`→127、それ以外→126、シグナル終了→128+sig)
- **ビルトインは親プロセスで実行**(fork しない)so that `exit`/将来の `cd` が親に効く: `pwd` / `echo`(`-n`)/ `env` / `exit`
- **libft 追加**: `ft_strdup` / `ft_strndup` / `ft_strncmp` / `ft_strchr`
- Makefile / libft Makefile: 新規ソースを SRCS に明示追記

### Verification(すべて実測 / Intel Mac・Apple clang 14・GNU Make 3.81)
- ✅ `make re` 警告ゼロ、二度目 `make` 再リンクなし
- ✅ `ls` / `/bin/ls -d /tmp`(絶対パス)/ `pwd` / `echo -n` / `env` / `exit` 動作
- ✅ 存在しないコマンド → stderr `command not found` + 終了コード **127**、非実行/ディレクトリ → **126**、`ls` → **0**
- ✅ 前後空白・空行で正常(クラッシュなし)
- ✅ `leaks` 自作リーク **0**(親、127経路含む。子の execve 失敗経路は free 後 exit を code review で担保)
- ✅ norminette: 42 ヘッダー以外の違反ゼロ(関数25行 / 5関数 / while のみ / グローバル0個 / 80列)
