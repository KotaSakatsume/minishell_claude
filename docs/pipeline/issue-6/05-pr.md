## PR Body

### Summary
シェル所有の可変 env を導入し、残りのコア組み込み cd / export / unset を実装する(Issue #6)。**`cd` で移動でき、`export`/`unset` で環境を変更できる。**

- **可変 env**: `t_shell.envp` → `char **env`。起動時に複製して所有(`env_dup`)、終了時に解放(`env_free`)。`env` builtin / PATH探索 / `execve` / cd・export・unset が**同一の可変コピー**を参照
- **env ヘルパ**: env_count/env_dup/env_get/env_free/env_unset(env_utils.c)、env_set(cpy/make_kv/env_find/env_append, env_set.c)。malloc 失敗時は**非破壊**
- **cd**(相対/絶対のみ): chdir + PWD/OLDPWD 更新。引数なしは no-op(HOME移動は別Issue)
- **export NAME=VALUE**: 追加/上書き、識別子バリデーション、`export` 単独は env 一覧、値なし NAME は no-op
- **unset NAME**: env から削除

### Verification(すべて実測 / Intel Mac・Apple clang 14・GNU Make 3.81)
- ✅ `make re` 警告ゼロ、二度目 `make` 再リンクなし、ヘッダ依存で再コンパイル
- ✅ `cd /tmp; pwd` / `cd ..` 反映、`cd /nonexistent` でエラー + カレント不変
- ✅ `export A=1`→`A=2` 上書き(重複なし)、`unset A` で消滅
- ✅ `export PATH=/bin` 変更が**外部コマンド解決に反映**(可変env一本化の主目的)
- ✅ `export ?bad=1` → `not a valid identifier` + status 1、env 不変
- ✅ 回帰: `ls` / `echo -n` / `env` / `exit` 従来どおり
- ✅ `leaks` 自作リーク **0**(env_dup/set 追加・上書き/unset/export 一時key/全エラー経路)
- ✅ norminette: 42 ヘッダー以外の違反ゼロ(関数25行 / 5関数 / while のみ / グローバル0個 / 80列)

### Known / Deferred
- **42 ヘッダー未付与**: norminette `INVALID_HEADER` のみ。42 ログイン確定待ちで保留。

- **nice**: `env_get` は将来の `$VAR` 展開/cd 拡張用に定義(現状未使用)。`export` 単独表示は最小形式。
